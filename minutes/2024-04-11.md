Continued the discussion, BBF API Review.
Discussion about the current IETF hardware modeling having leafs that are RO and not RW.  Concern related to how to have an intended configuration using the hardware model.  There is a concern that creating a similar model that includes RW leafs would not be in the spirit of NMDA.  Contributors are suggested to discuss with IETF and determine how to support using the hardware structure as a way to capture a preliminary structure and then validate against the actual structure.  For example have an intended configuration of a slot anticipating a particular type of plugable, and then check one the plugable is available.
